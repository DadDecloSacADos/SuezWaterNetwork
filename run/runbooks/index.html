Résolution incidents courants¶

Capteurs ne remontent pas de données¶

  • Vérifier connectivité réseau
  • Vérifier credentials connecteur PLTF
  • Vérifier logs gateway
  • Redémarrer service si nécessaire

Simulation EPANET timeout¶

  • Vérifier taille modèle (nombre nœuds)
  • Réduire durée timestep si trop grande
  • Checker ressources serveur (CPU, RAM)

Alerte faux positif pression¶

  • Ajuster seuils pression pour zone
  • Ajouter délai avant alerte
  • Valider capteur (calibration)
