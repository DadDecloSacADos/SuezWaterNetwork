Prérequis : Glossaire IWA | Hydraulique

Framework IWA¶

Qu'est-ce que le bilan IWA ?¶

L'International Water Association (IWA) définit un framework standard pour analyser où va l'eau dans un réseau d'eau potable.

Le bilan sépare :

VOLUME ENTRANT
    │
    ├─► CONSOMMATION FACTURISÉE (Billed Metered)
    │   ├─► Domicilaire facturisée
    │   └─► Non-domicilaire facturisée
    │
    ├─► CONSOMMATION NON FACTURISÉE (Unbilled Metered)
    │   ├─► Domicilaire (gratuité, usage public)
    │   └─► Non-domicilaire (gratuité)
    │
    └─► NON REVENUE WATER (NRW) / PERTES
        ├─► PERTES APPARENTES (Apparent Losses)
        │   ├─► Erreurs de mesure (compteurs)
        │   └─► Vols/usages non autorisés
        │
        └─► PERTES RÉELLES (Real Losses / Fuites)
            ├─► Ruptures conduites
            ├─► Fuites jonctions
            ├─► Fuites branchements
            └─► Débordements réservoirs

Indicateurs IWA (A, B, C, D)¶

Indicateur Formule Interprétation
A % NRW / Volume entrant Taux global de perte — % du volume perdu
B NRW / Longueur réseau / Jour Débit spécifique de fuite par km/jour — détecte surcharges
C % Pertes réelles / % Consommation facturisée Ratio perte réelle sur consommation — efficacité relative
D Nbr fuites / Nbr raccordements / An Intensité des défaillances — indice de détérioralité

Benchmark : - Pays développés : A = 10-15% (excellent) - Pays en développement : A = 30-50% (mauvais) - France (SUEZ) : A = 15-20% (bon)

Défis d'implémentation¶

🔴 Pourquoi c'est bloqué ?

  1. Spécification incomplète : Comment calculer exactement les "pertes réelles" ?
  2. Différence mesure - consommation connue ?
  3. Simulation EPANET vs réalité ?

  4. Extrapolation depuis détection de fuites ?

  5. Données manquantes : Certains clients n'ont pas :

  6. Compteurs à tous les nœuds
  7. Historique complet des fuites

  8. Enregistrements de consommation détaillés

  9. Validation des résultats : Comment savoir si les indicateurs sont corrects ?

  10. Pas de benchmark interne
  11. Dépend fortement des données saisies

  12. Erreurs de mesure masquent le bilan réel

  13. Cas d'usage multiples non couverts :

  14. Par DMA ? (zone)
  15. Par tronçon de réseau ?
  16. Comparaison inter-périodes ?
  17. Simulation prédictive ?

Configuration & Administration¶

Paramètres du bilan¶

Configuration de la période de calcul :
├─ Durée : Jour / Semaine / Mois / Trimestre / Année
├─ Date de début / fin
│
Configuration des sources de données :
├─ Source débit entrant (compteur ? simulation ?)
├─ Mapping consommation facturisée (source externe ?)
├─ Mapping pertes réelles (événements fuites ? algorithme ?)
│
Configuration d'export :
├─ Format (Excel ? PDF ?)
├─ Destinataire (rapports régulateurs ?)
└─ Fréquence automatique (mensuel ?)
