Capteurs & IoT¶

Objectif de cette page

Comprendre la gestion des capteurs IoT, les types de données collectées et l'intégration dans AQWN.

Types de capteurs¶

Capteur Mesure Localisation Fréquence
Débitmètre Débit (L/min, L/h) Entrée zone, tronçons critiques 1-5 min
Manomètre (Pression) Pression (bar) Nœuds clés réseau 1-5 min
Logger acoustique Bruit (dB, fréquence) Jonctions, vannes Continu
Capteur qualité Chlore, turbidité, pH Points critiques 1-15 min
Niveau réservoir Hauteur eau (m) Réservoirs 1-5 min
Température Température eau (°C) Points divers 15 min
AMR Consommation (m³) Compteurs clients Horaire/jour

Stockage séries temporelles¶

  • Format : Time Series Database (InfluxDB, Prometheus, etc.)
  • Granularité : Généralement 1-5 minutes
  • Rétention : 1-5 ans selon contrat
  • Accès : REST API pour requêtes analytiques

Exemple structure :

{
  "measurement": "pressure",
  "tags": {
    "zone": "DMA_01",
    "location": "node_42"
  },
  "fields": {
    "value": 2.45,
    "validity": true
  },
  "timestamp": "2024-04-09T14:30:00Z"
}

Connecteurs de données¶

Chaque source capteur nécessite un connecteur pour :

  1. Authentifier accès
  2. Parser le format natif
  3. Transformer en format AQWN
  4. Gérer la fréquence d'ingest

Connecteurs disponibles : - PLTF (remplacement progressif WN-spécifiques) - SEF (Sector Event Flow) - AMR (lecteurs compteurs) - Primayer (loggers acoustiques — 2024) - Geoserver (SIG) - APIs tierces

Gestion qualité données¶

Issue Action
Données manquantes Flag "invalide", alerte à l'opérateur
Valeurs aberrantes Détection outliers, flagging
Décalage temporel Synchronisation time servers
Perte connectivité Buffer local, retry
Capteur défaillant Historique panne, remplacement recommandé
