Hydraulique & Simulateur¶

Objectif de cette page

Comprendre le moteur EPANET, la simulation hydraulique, les scénarios et l'intégration avec les données du réseau.

Prérequis : Vue 360° | Glossaire hydraulique

Contexte métier¶

La modélisation hydraulique est le cœur technique d'AQWN. Elle permet de :

  • Simuler le comportement du réseau d'eau sous différentes conditions
  • Tester l'impact de modifications (fermeture de conduite, changement de pompe)
  • Estimer les pressions et débits en tout point du réseau
  • Valider la cohérence des mesures de capteurs en temps réel
  • Identifier les zones de faible pression ou surdimensionnées

Moteur EPANET — Intégration¶

Qu'est-ce qu'EPANET ?¶

EPANET est un simulateur hydraulique open-source de l'US EPA qui calcule :

  • Les pressions aux nœuds
  • Les débits dans les conduites
  • Les niveaux dans les réservoirs
  • Les vitesses d'écoulement

Intégration dans AQWN¶

AQWN encapsule EPANET via :

  1. Modèle hydraulique : Représentation du réseau (nœuds, conduites, pompes, vannes, réservoirs)
  2. Fichier INP : Format standard EPANET importé dans AQWN
  3. Conditions limites : Débits entrants, niveaux réservoirs, consommations
  4. Simulation pas à pas : Exécution en arrière-plan, stockage des résultats
  5. Visualisation : Affichage des résultats sur la cartographie

Sources de données d'entrée : - Capteurs de pression - Débits mesurés - Niveaux de réservoirs - Scénarios configurés par l'utilisateur

Résultats : - Séries temporelles de pression, débit, vitesse - Comparaison mesures réelles vs simulation - Identification de divergences (symptôme de fuite)

User Journey : Configuration du modèle¶

  1. Créer une entité "Modèle EPANET" avec :
  2. Nom et description
  3. Fichier INP

  4. Paramètres de simulation (timestep, durée max)

  5. Mapper les entités :

  6. DMA ↔ Nœud/Zone EPANET
  7. Capteurs réels ↔ Nœuds/conduites EPANET

  8. Pompes ↔ Vannes EPANET

  9. Configurer l'automatisation :

  10. Quand lancer les simulations (tous les jours ? à demande ?)
  11. Quels résultats stocker

User Journey : Exécution d'une simulation¶

  1. Création d'un scénario (depuis la carte ou interface)
  2. Isolation d'une conduite
  3. Modification d'un débit

  4. Ajustement d'une pompe

  5. Lancement de la simulation

  6. AQWN charge le modèle EPANET
  7. Applique les conditions limites
  8. Exécute la simulation pas à pas

  9. Stocke les résultats

  10. Analyse des résultats

  11. Comparaison avec la base scenario (mesures réelles)
  12. Visualisation sur la carte
  13. Graphiques superposés (simulation vs mesures)
  14. Export des données

Ganessa (moteur précédent)¶

Ganessa était le moteur hydraulique précédent. Il est progressivement remplacé par EPANET.

⚠️ Migration en cours : Certains documents anciens (2015-2018) référencent encore Ganessa sans noter la transition vers EPANET.

État : - Ganessa : Déprecié - EPANET : Nouveau standard (depuis ~2020) - Certains clients encore en Ganessa

Modèle de données hydraulique¶

Entité Description Exemple
Nœud (Node) Point du réseau (jonction, réservoir) Pression 3.5 bar
Conduite (Link) Tronçon entre deux nœuds Débit 50 L/s
Pompe Élément moteur Vitesse 1500 rpm
Vanne Élément de contrôle Ouverture 80%
Réservoir (Tank) Élément de stockage Niveau 12 m
Demande Consommation à un nœud 0.5 L/s/hab
