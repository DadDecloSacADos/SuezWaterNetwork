Incohérences & Divergences documentées¶

Problèmes de cohérence entre spécifications, code et backlog identifiés.

1. Ganessa vs EPANET — Double référence¶

Problème : - Docs récentes (2020+) favorisent EPANET - Docs anciennes (2015-2018) référencent Ganessa sans noter migration - Certains clients encore en Ganessa

Impact : Confusion sur "quel moteur utiliser"

À faire : - [ ] Audit complet : qui est encore en Ganessa ? - [ ] Timeline migration claire - [ ] Migration guide pour clients

2. WPF Configurateur vs HTML Prototype¶

Problème : - WPF = legacy, maintenance passive - HTML/Angular = prototype, viabilité incertaine - Double charge de travail pendant transition

Impact : Roadmap surchargée, délais rallongés

À faire : - [ ] Décision : HTML viable ? Quand EOL WPF ? - [ ] Plan consolidation UI - [ ] Priorité features : WPF ou HTML ?

3. IWA Water Balance — Spécification incomplète¶

Problème : - 106 items, 48% complétion (taux le plus bas) - 55+ US ouvertes - Spec v4 non finalisée - Calcul pertes réelles pas clairement défini

Impact : 🔴 CRITIQUE — Blocage tenders clients demandant bilan IWA

À faire : - [ ] Écrire spec v4 COMPLÈTE - [ ] Identifier source "pertes réelles" (simulation ? fuites détectées ?) - [ ] Validation avec expert métier + client pilote - [ ] Plan accélération Q2 2024

4. OpenLayers version — Specs vs backlog¶

Problème : - Quelques specs mentionnent "v8.x" - Backlog contains "upgrade v10.3" (2024) - Specs anciennes pas à jour sur version

Impact : Confusion sur capacités réelles (performance, features)

À faire : - [ ] Update documentation avec v10.3 features - [ ] Communiquer breaking changes clients

5. SEF vs PLTF Connectors — Frontière floue¶

Problème : - Docs 2015 : SEF = connecteur propriétaire événements - Roadmap 2024 : PLTF remplace connecteurs WN-spécifiques - Pas clair : SEF est-il "SEF Connector PLTF" ou separate ?

Impact : Implémentation intégrations confuse, dépendances mal définies

À faire : - [ ] Clarifier : SEF comme module métier VS SEF comme connecteur - [ ] Documenter intégration PLTF ↔ SEF - [ ] Roadmap remplacement SEF connecteur propriétaire

6. CWE-79 XSS — On Hold sans plan¶

Problème : - Vulnérabilité XSS détectée - Statut : On Hold (pas de plan de remédiation) - Risque sécurité data utilisateurs

Impact : Blocage potentiel déploiements clients security-focused

À faire : - [ ] Severity assessment complète - [ ] Plan remédiation : quand ? quelle effort ? - [ ] Workarounds courts terme si besoin

Résumé exécutif¶

Blocages critiques : 1. IWA Water Balance spécification (48% complétion) 2. WPF→HTML convergence design 3. CWE-79 XSS plan remédiation

Recommandations prioritaires : - Sprint dédié IWA spec clarification (1-2 semaines) - Architecture review WPF/HTML (décision go/no-go) - Security audit CWE-79 + plan remédiation
