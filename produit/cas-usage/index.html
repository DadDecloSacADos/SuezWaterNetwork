Cas d'usage — Aquadvanced WaterNetworks¶

Cas 1 : Détection rapide fuite (Cas principal)¶

Scénario : Réseau perd ~20% débit sans cause apparente.

  1. AQWN détecte baisse pression anormale dans zone
  2. Data Correlator analyse données capteurs
  3. Loggers acoustiques confirment bruit fuite
  4. LLI localise fuite à rue précise
  5. Système SEF active alerte opérateur
  6. Opérateur envoie équipe → fuite réparée en 2h (vs 1-2 semaines sans AQWN)

Valeur : Réduction pertes -15%, coûts intervention diminués.

Cas 2 : Validation simulation vs réalité¶

Scénario : Manager veut savoir si réseau fonctionne comme prévu.

  1. AQWN simule réseau EPANET avec débits actuels
  2. Compare résultats simulation vs mesures réelles capteurs
  3. Identifie zones divergences (symptôme fuite ou mesure erronée)
  4. Rapport mensuel montre état hydraulique

Valeur : Compréhension profonde comportement réseau, base de décisions.

Cas 3 : Rapport conformité régulateur¶

Scénario : Autorité demande bilan eau standard IWA.

  1. AQWN calcule automatiquement bilan period (mois/an)
  2. Indicateurs A/B/C/D générés
  3. Comparaison période précédente
  4. Rapport PDF exporté certifié

Valeur : Conformité légale, transparence.

Cas 4 : Configuration nouvelle zone¶

Scénario : Nouveau DMA sur réseau existant.

  1. Admin ajoute entité DMA + conduites/nœuds
  2. Configure capteurs (pression, débit)
  3. Importe modèle EPANET zone
  4. Défini seuils alertes, secteurs
  5. Zone opérationnelle

Valeur : Déploiement rapide.

Cas 5 : Prédiction détérioration¶

Scénario : Manager veut anticiper ruptures futures.

  1. Historique fuites analysé (type, localisation, âge conduite)
  2. Tendance extraction (ex : +3 fuites/an)
  3. Prédiction ruptures probables

Valeur : Planification maintenance préventive.
