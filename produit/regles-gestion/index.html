Règles de gestion — AQWN¶

Gestion des données¶

  • Saisie manuelle : Validée avant enregistrement
  • Import automatique : Via connecteurs PLTF, transformation standard
  • Archivage : Données > 5 ans compressées
  • Correction : Historique traçabilité conservé

Gestion des entités¶

  • DMA : Identifiant unique, nom, limites géo
  • Secteur : Sous-ensemble DMA, fermeture possible
  • Capteur : Localisation, type, calibration requise
  • Fuite : Minimum 1 date + 1 localisation

Gestion des alertes¶

  • Seuil minim pression : 1.5 bar (configurable)
  • Seuil maxim pression : 5 bar (configurable)
  • Délai avant alerte : 15 min (ignore spikes transitoires)
  • Escalade : Notification → Action automatique (selon config)

Gestion des droits¶

  • Admin : Configuration + gestion utilisateurs
  • Opérateur : Consultation + création événements
  • Lecteur : Consultation readonly

Cycle de vie fuite¶

Détectée → Validée → Réparée → Fermée

Chaque transition peut être forcée manuellement par opérateur.

Période de calcul¶

  • Jour : 00:00-23:59 (fuseau horaire local)
  • Mois : 1er-dernier jour
  • Année : 1er jan - 31 dec
